Eternal Linux can be considered lightweight in philosophy, but not in the same way as distros like Alpine or Arch minimal installs

Eternal Linux and Lightweight Characteristics
DIY Base: Eternal Linux doesn’t ship its own ISO — you start from Fedora and strip/reshape it into Eternal Linux. This means you can choose a minimal Fedora Custom Operating System install, which keeps the system lean.

Customizable: Because you decide what packages to install, Eternal Linux can be as lightweight or as heavy as you make it. If you only install the essentials, it runs with a small footprint.

RPM Ecosystem: It inherits Fedora’s RPM package management, which isn’t inherently lightweight compared to distros like Alpine (musl + BusyBox), but it’s stable and flexible.

Branding Layer: Eternal Linux adds branding, configuration, and security hardening — these don’t add much overhead, so the system remains close to Fedora’s baseline performance.

Comparison
Alpine Linux → Ultra‑lightweight, musl libc + BusyBox, designed for containers and embedded systems.

Debian Minimal / Arch Minimal → Lightweight if you install only the base system.

Eternal Linux → Lightweight if you choose minimal Fedora base, but not inherently ultra‑minimal. It’s more about security and independence than extreme resource savings.
